# 프로젝트 II: 대상 시스템 품질 요소 추정서

'MiniDrive' 클라우드 파일 관리 시스템의 핵심 기능과 요구사항을 바탕으로, 주요 이해관계자별 품질 기대치를 추정하고 관리 우선순위를 정리한 것입니다.

## 1. 사용자(User) 측면의 품질 기대치
사용자는 시스템을 직접 활용하여 업무 생산성을 높이고자 하는 200여 명의 조직원 및 외부 협력사입니다.

* **신뢰성 (Reliability)**: 업로드한 파일이 손상되지 않고 언제 어디서든 정확하게 다운로드되어야 하며, 데이터의 정합성이 보장되어야 합니다
* **효율성 (Efficiency/Performance)**: 200명의 동시 접속 환경에서도 파일 검색 및 다운로드 속도가 저하되지 않고 빠르게 동작해야 합니다
* **사용 용이성 (Usability)**: 별도의 교육이나 프로그램 설치 없이 웹 브라우저를 통해 직관적으로 파일을 업로드하고 관리할 수 있어야 합니다
* **가용성 (Availability)**: 업무 시간 내에 시스템 중단 없이 안정적으로 접근 가능해야 하며, 실수로 삭제한 파일도 복구할 수 있는 안정 장치가 필요합니다

## 2. 개발자 및 유지보수 담당자 측면의 품질 기대치
시스템을 설계, 구현하고 향후 기능을 확장해야 하는 엔지니어 관점의 품질 요소입니다.

* **유지보수성 (Maintainability)**: 향후 '문서 공동 편집'이나 'AI 기반 검색' 등 추가 기능 확장을 고려하여 코드가 모듈화되고 구조화되어야 합니다
* **이식성 (Portability)**: 다양한 웹 브라우저 환경에서 동일하게 동작해야 하며, 특정 OS나 환경에 종속되지 않는 웹 표준을 준수해야 합니다
* **검증 가능성 (Testability)**: 권한 제어(RBAC) 및 암호화 전송 등 복잡한 보안 로직이 설계대로 작동하는지 쉽게 테스트하고 검증할 수 있어야 합니다
* **재사용성 (Reusability)**: 파일 업로드 모듈이나 검색 엔진 인터페이스 등을 향후 다른 사내 프로젝트에서도 활용할 수 있도록 범용적으로 설계해야 합니다

## 3. 관리자(Manager) 및 조직 측면의 품질 기대치
비용, 보안, 자원 관리 등 프로젝트의 성공적인 운영과 리스크 관리를 중시하는 관점입니다.

* **보안성 (Security)**: 조직의 핵심 자산인 데이터 유출을 방지하기 위해 로그인 인증, 권한 기반 접근 제어, 데이터 암호화가 철저히 이루어져야 합니다
* **확장성 (Scalability)**: 조직 규모가 200명 이상으로 커지거나 데이터 사용량이 급증할 경우, 서버 자원을 효율적으로 증설하고 관리할 수 있어야 합니다
* **추적성 (Traceability)**: 외부 공유 링크의 사용 이력이나 사용자별 저장 공간 사용 현황을 실시간으로 모니터링하고 관리할 수 있어야 합니다
* **규정 준수 (Compliance)**: 외부 협력 업체와의 공유 시 설정된 만료 기간을 엄격히 준수하여 보안 사고 리스크를 최소화해야 합니다

## 4. 품질 우선순위 요약 (Priority Matrix)

| 우선순위 | 품질 요소 | 사유 |
| :--- | :--- | :--- |
| **High** | **보안성** |조직 내부의 중요 데이터 유출 방지를 최우선으로 고려햐아함 |
| **High** | **성능(효율성)** |200명 동시 사용 시 속도 저하는 업무 마비로 이어질 수 있음 |
| **Medium** | **신뢰성** | 버전 관리 및 복구 기능을 통한 데이터 무결성 유지 |
| **Medium** | **확장성** | 향후 공동 편집 및 AI 기능 도입을 위한 구조 준비 |
